# Agent Rules (NeosTech)

## Workflow
- Do NOT modify files unless I say the exact word: **APLICA**
- First deliver: (1) plan, (2) unified diff, (3) list of files to touch, (4) expected `git diff --stat`.
- Max **3 files per PR**.
- No "format all". No refactors outside the requested scope.
- Always give exact paths.

## Git discipline
- One task = one branch.
- Keep commits small and scoped.
- Never commit secrets, tokens, or `.env` values.

## Output format
- Prefer unified diffs for code changes.
- If something is ambiguous, ask max 3 questions.
